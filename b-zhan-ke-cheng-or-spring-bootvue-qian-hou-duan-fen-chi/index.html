· 2021-12-21 ·

昨天和同事们一起聊天，小吴同学吐槽起前东家奇葩，这年头用的还是古老难用的JSP，来了个实习生看完代码连夜离职。一直知道前后端分离这个概念，学过JAVA，但！对于背后的技术实现和演变一直不太清楚，干脆乘机了解一下咯，传送门：4小时入门Spring Boot +Vue前后端分离

为什么要前后端分离？

传统的Java Web开发中，前端使用JSP开发, JSP 不是由后端开发者来独立完成的。
前端-->HTML静态页面-->后端- JSP，开发方式效率极低。

什么是前后端分离？

前后端分离就是将⼀个单体应⽤拆分成两个独⽴的应⽤，前端应用+后端应用。

  • 前端应用: 负责数据展示和用户交互，独立编写客户端代码

  • 后端应用: 负责提供数据处理接口，独立编写服务端代码

  • 前端通过Ajax请求来访问后端的数据接口，将Model展示到View中。前端应⽤和后端应⽤以JSON格式进⾏数据交互。

  • 流程：前端HTML -->Ajax --> RESTful 后端数据接口。

前后端开发者只需要提前约定好接口文档(URL、 参数、数据类型..)，然后分别独立开发即可，前端
可以造假数据进行测试，完全不需要依赖于后端，最后完成前后端联调集成即可，真正实现了前后端应用的
解耦合，极大地提升了开发效率。

*补充几个概念

单页web应用（single page web application，SPA），就是只有一张Web页面的应用。单页应用程序 (SPA) 是加载单个HTML 页面并在用户与应用程序交互时动态更新该页面的Web应用程序。浏览器一开始会加载必需的HTML、CSS和JavaScript，所有的操作都在这张页面上完成，都由JavaScript来控制。因此，对单页应用来说模块化的开发和设计显得相当重要。

vue-router是Vue.js官方的路由插件，它和vue.js是深度集成的，适合用于构建单页面应用。vue的单页面应用是基于路由和组件的，路由用于设定访问路径，并将路径和组件映射起来。传统的页面应用，是用一些超链接来实现页面切换和跳转的。在vue-router单页面应用中，则是路径之间的切换，也就是组件的切换。路由模块的本质 就是建立起url和组件之间的映射关系。

Vue + Element UI

Element是饿了么的组件库，关于这个看官方文档就够了传送门：Element UI 官方文档

UP主讲了一些Element UI后台管理系统菜单的主要标签:
●el-container: 构建整个页面框架。
●el-aside:构建左侧菜单。

●el-menu: 左侧菜单内容,常用属性:
:default-openeds:默认展开的菜单，通过菜单的index值来关联。
:default-active:默认选中的菜单，通过菜单的index值来关联。
●el-submenu: 可展开的一级菜单，常用属性:
index:菜单的下标，文本类型，不能是数值类型。
●el-menu-item: 菜单的子节点，可以直接点，不可再展开，常用属性:
index:菜单的下标，文本类型，不能是数值类型。

●template: 对应el-submenu的菜单名。
●i:设置菜单图标，通过class属性实则，例如：
。el-icon-messae
。el-icon-menu
。el-icon-setting
